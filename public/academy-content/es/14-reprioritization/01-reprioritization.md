# Repriorización y Re-edición

Incluso después de que un proyecto haya sido lanzado, la agilidad es clave. El flujo de **Repriorización** permite realizar cambios controlados en el plan del proyecto.

## 1. El Proceso

1. **Solicitud:** Un miembro del equipo señala la necesidad de realizar un cambio.
2. **Desbloqueo:** El Administrador concede explícitamente acceso al **Modo de Repriorización** para el proyecto.

![User Management section](/academy-screenshots/admin-panel/user-management.png)
*Ir a la sección Gestión de Usuarios para localizar al miembro del proyecto.*

![Project Access section](/academy-screenshots/admin-panel/project-access.png)
*Conceder acceso específico al proyecto en "Modo de Repriorización" para habilitar capacidades de edición.*

3. **Edición:** Los usuarios seleccionados pueden modificar clasificaciones, agregar nuevos elementos o archivar los obsoletos.
4. **Re-finalizar:** El proyecto pasa por un proceso de aprobación simplificado para volver al estado **"Live"**.

## 2. ¿A Quién se le Puede Conceder Acceso?

Los administradores pueden conceder derechos de edición con diferentes niveles durante esta fase:

* **Acceso Completo de Edición:** Puede agregar o eliminar elementos (normalmente los responsables).
* **Acceso de Reordenación:** Solo puede ajustar las prioridades de los elementos existentes (normalmente los principales stakeholders).

![Access Management section](/academy-screenshots/admin-panel/access-management.png)
*Seleccionar el nivel de permiso adecuado (Edición Completa o Solo Reordenar) para cada stakeholder.*

Esto garantiza que el proyecto permanezca estable mientras se adapta a nueva información.
