# Access Management

The **Access Management** tab in the Admin Panel provides a centralized view of all special permissions granted to collaborators across your businesses. It allows Admins to monitor who has edit or rerank access and revoke it when no longer needed.

### Overview Dashboard

At the top of the Access Management page, you will find four metric cards:

*   **Total Users with Access**: Number of distinct users who have at least one special permission.
*   **Reranking Access**: Total instances of reranking access granted.
*   **Project Edit Access**: Total instances of project edit access granted.
*   **Active Permissions**: Total count of all active special permissions.

![Access Management Dashboard](/academy-screenshots/projects/access-management.png)

### Viewing and Filtering Access

1.  **Select Business**: Use the dropdown to switch between different businesses. The table will update to show only access records for the selected business.
2.  **Access List**: The table displays:
    *   **User**: Name and email of the collaborator.
    *   **Permissions**: Badges indicating if they have **Rerank** or **Edit** access.
    *   **Projects with Access**: List of specific projects they can edit.

### Revoking Access

If a collaborator no longer requires special access, you can revoke it directly from the table:

*   **Revoke Rerank/Edit**: Click the specific action icon (Lock or Trash icon) in the **Actions** column to revoke a single permission.
*   **Revoke All**: Use the **Revoke All** button to remove all special permissions for that user on the selected business.

Revoking access takes effect immediately, and the collaborator will return to the standard read-only view for launched projects.
