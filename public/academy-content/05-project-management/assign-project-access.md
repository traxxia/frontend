# Assigning Project Access

Once a project has been **Launched**, it enters a read-only state to preserve the integrity of the prioritization and execution phase. However, there are scenarios where an Admin may need to grant temporary or specific access for a collaborator to **Rerank** or **Edit** a launched project.

### How to Assign Access

To grant special access to a collaborator:

1. Navigate to the **Admin Panel** from the main navigation.
2. Select the **User Management** tab.
3. Locate the user you wish to grant access to.
4. Click the **Project Access** button located at the top right of the user management section.

![Project Access Button](/academy-screenshots/projects/project-access1.png)

### The Add Project Access Modal

When you click the button, a popup will appear with several configuration options:

- **Access Type**: Choose between **Enable Reranking Project** (allows the collaborator to change the project's priority) or **Edit the Project** (allows modification of project details).
- **Business**: Select the specific business where the project resides.
- **Project**: If "Edit the Project" is selected, choose the specific project from the dropdown.
- **Collaborators**: Select one or more collaborators who should receive this access.

![Add Project Access Modal](/academy-screenshots/projects/project-access.png)

Once you click **Continue** and confirm the details, the selected collaborators will immediately have the requested access.
