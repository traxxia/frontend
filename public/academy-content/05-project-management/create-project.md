# Creating New Projects

While most projects originate from the **Kickstart Process**, you can also create standalone projects directly within the Projects Section.

## 1. The "New Project" Button
The **New Project** button is located in the top-right corner of the Projects dashboard. Clicking this button opens a modal where you can define the core parameters of your initiative.

![New Project Button](/academy-screenshots/projects/new-project-button.png)

## 2. Initial Setup
When creating a project, you'll provide:
- **Project Name**: A clear, identifiable title.
- **Status**: By default, new projects are set to **Draft**.
- **Collaborators**: Assign team members who will participate in ranking and execution.

## 3. Saving as Draft
Once saved, the project immediately appears in the dashboard in the **Draft** status. It is now ready for detailed refinement by the project owner or admin before it joins the ranking workflow.

> [!TIP]
> Projects created manually follow the same collaborative rules as kickstarted projects—they require ranking and team input before they can be officially launched.
